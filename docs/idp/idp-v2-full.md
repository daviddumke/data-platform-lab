# IDP v2 — Master Index · Senior Data Engineer Roadmap

> **What this document is.** A navigation surface for the Senior-calibrated Data Engineering Individual Development Plan (IDP v2). The substantive content lives in 27 per-Epic pilot files — each one Jira-ready, each one structured around competency sub-skills with multi-level checkbox progressions targeting Senior calibration.
>
> **How the IDP is shaped.**
> - 27 Epics → ~126 Stories → ~285 Senior-targeted sub-skills.
> - Each sub-skill = Title (Senior action-verb) + Target line + Activity & Artifact paragraph (Senior DoD) + 5-level checkbox progression (Junior → Mid → Senior 🎯 → Staff → Principal).
> - A Skill is mastered when all its 🎯 sub-skill levels have been reached. A Set of Skills is mastered when all its skills are mastered.
> - 9 industry certifications mapped across Epics 19–24.
> - Roadmap calibrated to **Senior**, with selected Staff/Principal stretch sub-skills marked explicitly.

│   ├── rfcs/                    # Request for Comments

## ⏱ Pacing Cadences

Pick one — calibrated to ~285 Senior sub-skills:

| Cadence | Pace | Time to complete |
|---|---|---|
| Intensive | 2 sub-skills / week | ~3 years |
| Sustainable | 1 sub-skill / week | ~5–6 years |
| Per-Epic focal | 1 Epic / 2 months | ~4–5 years |
| Cert-driven | 1 cert / 6 months | ~5 years for all 9 certs (Epics 19–24) + fundamentals in parallel |

## 🗺 Next Steps

1. **Walkthrough.** Read each per-Epic pilot in order — mark already-mastered sub-skills (tick the 🎯 box), adjust Stories where the framing doesn't match the engineer's reality.
2. **Lab setup.** Initialize the `data-platform-lab` repository with the structure above. The minimum **walking skeleton** (Epic 25.3.1) = Compose + 1 source + 1 transformation + 1 DAG.
3. **Materialize in Jira.** Use `/create-issue` or `mcp__jira__jira_batch_create_issues` to create Epic + Stories + Sub-skills with hierarchy + labels. Each sub-skill section in a per-Epic pilot file is already shaped as a copy-pasteable Jira issue (Title + Target label + Activity & Artifact paragraph + checkbox progression).
4. **Versioning.** Once the engineer has a `data-platform-lab` repo, this index + pilots live at `docs/idp/` inside it, with diffs over time recording the evolution of the personal roadmap.

## 🔮 v3 Candidates (Future Additions)

Two emerging domains were deliberately left out of v2 and earmarked for v3:

- **Epic 28 — MLOps overlap with DataOps** (for DEs who collaborate directly with ML): feature engineering pipelines, model versioning (MLflow, Weights & Biases), drift detection, model serving infrastructure, A/B testing infrastructure, training/inference data parity, vector DB ops for RAG.
- **Epic 29 — AI/LLM for Data Engineering** (using AI *to build* data platforms): LLM-assisted SQL/code generation, embeddings pipelines, RAG over data catalogs, semantic layer + LLM Q&A, automated documentation, anomaly detection with LLMs, Text-to-SQL gateways.

These domains evolve rapidly in 2025–2026; the framing of "what's load-bearing" vs "what's hype" shifts month-to-month. They will be anchored once stable practices crystallize.
